## 第 3 个月：把你自己的方法量化化（关键章节）

**核心目标**：这是你最独特的资产——把你已有的赚钱方法变成可量化、可回测、可优化的系统。

### 周 1：写下你的「投资说明书」

不写代码，只写文档。强制自己把以下问题答清楚：

1. 我现在赚钱的方法，具体是什么？（用一段话描述）
2. 我会买什么样的股票？（具体到指标）
3. 我什么时候买入？（进场触发条件）
4. 我什么时候卖出？（止盈、止损、换股的条件）
5. 我的仓位怎么管理？（满仓？分批？）
6. 过去 3 年我大概的真实收益是多少？（如果有记录）
7. 我赚钱主要靠什么？（选股？择时？行业判断？）
8. 我亏过最大的钱是怎么亏的？

**交付物**：一份「我的投资方法论」文档（`docs/methodology.md`），至少 2000 字。

**这一步的价值**：你会发现，很多你以为清晰的规则，实际上是模糊的直觉。把它们写下来，是量化化的第一步。

### 周 2–3：把规则量化化

任务：把上面文档里的每条规则，变成可计算的指标。

举例（假设你说「我买景气度向上的低估值股票」）：

- 景气度向上 → 量化为：
  - 行业过去 N 季度营收同比 > 0？
  - 行业 PE 分位数 < 50%？
- 低估值 → 量化为：
  - PE TTM < 行业中位数？
  - PB < 历史 5 年的 30% 分位？
  - 或综合估值打分前 30%？

**关键挑战：数据**。

- 行业数据：tushare 有，可能要付费
- 财务数据：tushare 基础版有
- 估值数据：tushare 有 daily_basic 接口

如果某些数据拿不到，先简化规则用能拿到的数据近似。

**交付物**：一份「规则 → 量化指标」的对照表（`docs/rules_to_metrics.md`）。

### 周 4：回测你自己的方法

任务：严格按你写下的规则，在历史数据上模拟。

至少包括：

- 每月初按规则筛选股票
- 等权或按某种规则配置仓位
- 记录每笔交易
- 跟沪深 300 对比

**预期发现**（很多人不愿面对的真相）：

1. 你的真实 alpha 可能比你以为的小——可能你跑赢指数 5 个点，而不是 20 个点
2. 你赚钱的真实归因可能让你意外——你以为是选股能力，可能其实是行业 beta
3. 你的方法在某些市场环境下表现极差——可能你只在结构性牛市能赚钱

这一步会很扎心，但极其有价值。

### 月度交付物

- 「我的投资方法论」文档（2000+ 字）
- 「规则 → 指标」量化对照表
- 完整的「自己方法」回测 notebook
- 一份自我归因报告：「我真的有 alpha 吗？有多少？来源是什么？」

### 可衡量标准

- [ ] 你的方法被完整地量化化了（没有「凭感觉」的步骤）
- [ ] 你能客观评估自己的真实业绩（扣除 beta 后的 alpha）
- [ ] 你识别出了自己方法的至少 2 个弱点

## 第 4 个月：补足风控系统

**核心目标**：针对你说的「风控不太行」，建立量化风控体系。

### 周 1：仓位管理理论 + 实操

学习并实现：

1. **固定仓位**（基线）：每只股票等权
2. **波动率倒数仓位**：波动大的少买，波动小的多买
3. **风险平价**：每个标的对组合贡献的风险相等
4. **凯利公式（简化版）**：根据胜率和盈亏比决定仓位

任务：把你第 3 个月的策略，套用这 4 种仓位管理，对比表现。

**预期发现**：不同的仓位管理方法，可能让你的最大回撤从 -30% 降到 -20%，而收益几乎不变。这就是风控的价值。

### 周 2：止损规则

学习并实现：

1. 固定百分比止损（从买入价亏 N% 出场）
2. 跟踪止损（从最高价回撤 N% 出场）
3. 波动率止损（从买入价亏 N 倍 ATR 出场）
4. 时间止损（持有 N 天后无论盈亏出场）

任务：在你的策略上加上各种止损，看哪种效果最好。

**关键洞察**：止损会降低收益、降低回撤。问题是收益降多少、回撤降多少——找到最优平衡点。

### 周 3：分散化和组合层面风控

学习并实现：

1. 行业分散：同一行业最多持有 N 只
2. 个股集中度限制：单只股票最多占 N%
3. 总仓位控制：根据市场状态动态调整总仓位（如沪深 300 在 200 日均线下方时减仓）

任务：把这些规则加到你的策略上，看组合层面的最大回撤变化。

### 周 4：回撤控制和资金曲线管理

学习并实现：

1. 回撤触发减仓：策略累计回撤 > N% 时减仓 50%
2. 波动率目标：维持组合年化波动率在某个目标值
3. 资金曲线均线：策略净值跌破自己的 60 日均线时空仓

任务：实施这些规则，对比「裸策略」和「带风控策略」的指标。

### 月度交付物

- 4 个风控模块的完整实现
- 一份「风控前后对比报告」：展示风控对收益、回撤、夏普的影响
- 一份「我的风控规则」文档（`docs/risk_rules.md`）：固化下来的风控参数

### 可衡量标准

- [ ] 你的策略最大回撤显著降低（至少降低 30%）
- [ ] 夏普比率显著提升（至少提升 30%）
- [ ] 你能向别人解释每条风控规则的逻辑和成本

## 第 5 个月：多标的组合 + 因子初步

**核心目标**：从「单标的策略」升级到「组合策略」，理解 alpha 的横截面属性。

### 周 1：多标的数据准备

任务：

- 把数据扩展到沪深 300 全部成分股
- 处理大量数据的内存问题（分块、Polars、parquet）
- 处理停牌、ST、退市等特殊情况
- 建立「股票池」概念

### 周 2–3：简单因子选股

任务：实现一个简单的多因子选股策略：

- 计算 3–5 个因子（动量、价值、质量、波动率等）
- 横截面标准化（z-score）
- 等权合成总分
- 每月底选前 30 名，等权持有，月初再平衡

**关键挑战**：

- 横截面思维（一天看所有股票，而不是一只股票看所有天）
- 数据对齐（不同股票的交易日可能不一致）
- 处理新股、退市股

### 周 4：严格的样本外测试

**这是这个月最重要的环节，核心是避免过拟合。**

任务：

1. 把数据切分：
   - 训练集：2018-01 到 2022-12
   - 测试集：2023-01 到 2025-12
2. 在训练集上选因子、定权重、调参
3. 完全不要看测试集，直到最后一步
4. 用最终参数跑测试集，看表现差距

**预期发现**：训练集夏普 1.5，测试集夏普 0.5，这是过拟合的典型症状。

这一周的教训会让你受益终身——你会真正理解「回测美如画，实盘亏成狗」的根本原因。

### 月度交付物

- 多标的数据基础设施
- 一份多因子选股策略的完整实现
- 一份样本外测试报告：训练集 vs 测试集对比

### 可衡量标准

- [ ] 能流畅处理 300 只股票的数据（内存不爆）
- [ ] 实现了完整的「因子计算 → 横截面排序 → 选股 → 再平衡」流程
- [ ] 你亲身经历了一次过拟合，并理解了它的本质

## 几条贯穿 6 个月的原则

1. **每个 notebook 顶部写清楚**：研究问题、结论、关键参数。3 个月后回头看不会懵。
2. **结果不重要，过程重要**：你不是在找「赚钱的策略」，而是在建「研究能力」。
3. **诚实地面对失败的回测**：90% 的策略不行，这是常态。失败的研究也是研究。
4. **不要追求工具的完美**：工具是手段，研究是目的。够用就行，有需求再改。
5. **每月写一次月报**（放 `reports/`）：回顾这个月做了什么、学到了什么、下个月做什么。这一个动作让你的进度清晰可视。
6. **保持节奏**：周末或晚上 5–10 小时/周，够用。不要冲刺，不要中断。量化是马拉松不是短跑。
